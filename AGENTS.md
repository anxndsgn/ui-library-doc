# AGENTS.md

This file provides guidance to agents when working with code in this repository.

## Project Overview

A component registry and documentation site built with TanStack Start, React 19, Tailwind CSS v4, and shadcn registry system. Uses fumadocs-mdx for MDX content and file-based routing via TanStack Router.

## Commands

- `pnpm dev` — Start dev server (port 3000)
- `pnpm build` — Production build
- `pnpm lint` — Run oxlint
- `pnpm format` — Format with oxfmt (auto-fix)
- `pnpm format:check` — Check formatting without writing
- `pnpm typegen` — Regenerate fumadocs-mdx types (also runs on postinstall)
- `pnpm registry:build` — Build registry JSON to `public/r/` (run after adding/changing components)

## Generated Files — Do Not Edit

- `.source/` — Generated by fumadocs-mdx (run `pnpm typegen` to regenerate)
- `src/routeTree.gen.ts` — Generated by TanStack Router (auto-updates on dev server)
- `public/r/` — Generated by `pnpm registry:build`

## Adding a New Component (Registry Workflow)

1. Create component in `registry/default/ui/<component-name>.tsx`
2. Create demo in `registry/default/examples/<component-name>-demo.tsx`
3. Add entry to `registry.json`
4. Run `pnpm registry:build`
5. Write documentation page at `content/docs/components/<component-name>.mdx`

## Path Aliases

- `@/*` and `~/*` → `./src/*`
- `collections/*` → `./.source/*` (fumadocs generated types)
- `registry/*` → `./registry/*`

## Styling

- Tailwind CSS v4 — theme defined in `src/styles/app.css` using `@theme inline` (no tailwind.config file)
- shadcn CSS variables are the source of truth: `--background`, `--foreground`, `--card`, `--primary`, `--secondary`, `--muted`, `--accent`, `--border`, `--input`, `--ring`, etc.
- Base color is neutral (`components.json` → `tailwind.baseColor: "neutral"`); prefer standard shadcn classes like `bg-background`, `text-foreground`, `bg-card`, `text-muted-foreground`, `border-border`, `bg-primary`, `text-primary-foreground`, `bg-accent`, and `text-accent-foreground`
- Dark mode via `data-theme="dark"` attribute on `<html>`
- Use `cn()` utility from `@/lib/utils` for conditional class merging (clsx + tailwind-merge)

## Component Patterns

- shadcn style: `new-york` variant, CSS variables enabled, no RSC (`"rsc": false`)
- Plain function components (no forwardRef)
- Use Zod v4 for schema validation
- MDX source imports use `?raw` suffix for code display

## Commit Style

Use Conventional Commits: `feat:`, `fix:`, `chore:`, `docs:`, etc. All in English.
